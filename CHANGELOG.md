## 1.2.2

- **性能优化**：新增 `session_stats` 汇总表与 SQLite trigger，`get_sessions` 与 `get_history` 的 COUNT 查询改为按需计算（`include_total` 参数），大幅降低大表扫描开销；批量写入拆分为小批次提交，避免长 SQLite 写锁。
- **前端渲染优化**：消息组与系统消息添加 `content-visibility: auto` 与 `contain-intrinsic-size`，减少不可见区域的样式计算与重排。
- **安全修复**：统一 CQ 码参数转义方法 `_escape_cq_param`，修复媒体 URL 中 `&`、`[`、`]`、`,` 等特殊字符的逃逸问题。
- **数据准确性**：排行与统计查询排除 `user_id='0'` 的无效记录以及已撤回消息；上下文消息列表排序修正（由升序改为降序后再反转，确保返回顺序正确）。
- **锁资源管理**：下载锁引入引用计数机制，修复并发场景下的锁泄漏问题。
- **稳定性增强**：WebUI standalone 模式启动时自动初始化数据库，确保 schema 最新；认证接口增加请求体校验与 API Key 未配置时的明确 503 响应。

## 1.2.1

- WebUI 支持展示 CQ JSON 链接分享卡片的具体平台与标题，兼容 QQ 小程序、结构消息以及常见 CQ/HTML 转义格式。
- 新增 JSON 消息组件归档序列化，避免链接分享卡片在写入历史记录时丢失原始结构。
- 为 WebUI 静态资源增加版本参数，降低浏览器缓存导致前端更新不生效的概率。

## 1.2

- 优化 WebUI 图片消息展示，按原始图片比例分配更合适的前端显示尺寸。
- 修复媒体消息边框、hover 状态与快速滚动时的闪烁/遮罩观感问题。
- 改进向上加载历史消息时的滚动锚定，降低加载未缓存消息块造成的画面跳动。
- 优化左侧会话栏与成员列表动画，减少展开多人预览时的卡顿和头像后加载割裂感。
- 成员列表支持按会话全局搜索，并为左侧成员列表和右侧活跃排行拆分分页状态。
- 右侧数据分析栏按视口高度预估首屏排行数量，减少底部留白和延迟补载。
- 修复活跃成员排行/成员接口在同一用户多昵称场景下取名不稳定的问题。
- 增加图片尺寸解析与成员接口昵称稳定性的单元测试。
